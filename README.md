# Methods for Multi-Omics Integration

> A practical, reproducible tutorial collection that runs the major families of multi-omics integration methods end-to-end on a single public dataset.

## What this is

This repository is the **supplementary material** to the methodological review *"A practical review of multi-omics integration methods"* (Rebellon-Sanchez, in preparation) and to Chapter 1 of the corresponding doctoral thesis at the London School of Hygiene & Tropical Medicine. The review can be read on its own. This repository exists for the reader who finishes the review and asks the next, very reasonable question: *"all right, but how do I actually run any of this on my own data?"*.

Rather than answering that question with abstract code snippets, the seven tutorials here run the major families of multi-omics integration methods **end-to-end on a single public dataset** (MLOmics GS-BRCA, derived from TCGA), so the reader can compare what each method does to the same patients with their own eyes.

The tutorials are designed for PhD students, bioinformaticians and clinical researchers who have a basic R or Python background but have never touched multi-omics integration. We define every acronym, every metric and every output the first time it appears, and we walk the reader from one section to the next instead of dropping them into a chunk of code without context.

## Tutorials at a glance

| #   | Method      | Family                            | Library / language        |
|-----|-------------|-----------------------------------|---------------------------|
| 01  | Dataset     | n/a                               | R                         |
| 02  | sPLS-DA     | Sparse latent variable, single-block primer | `mixOmics` (R)   |
| 03  | DIABLO      | Sparse latent variable, multi-block | `mixOmics` (R)         |
| 04  | MOFA2       | Bayesian latent factor             | `MOFA2` (R + Python)     |
| 05  | SNF         | Network-based, patient similarity  | `SNFtool` (R)            |
| 06  | WGCNA       | Network-based, feature co-expression | `WGCNA` (R)            |
| 07  | VAE         | Deep generative                   | `torch` (Python via `reticulate`) |
| 08  | Comparison  | Side-by-side benchmark across all six methods | R               |

Each tutorial lives in its own folder and follows the **same nine-section template** so that you can cross-compare:

1. **Where this method fits** in the multi-omics landscape
2. **The biological problem** it solves
3. **Intuition before any equation**
4. **Mathematical formulation**
5. **Toy example by hand**
6. **Library requirements** (data format, sample/feature orientation, transformations)
7. **Canonical workflow** using the library's documented API
8. **Reading the outputs** (every metric and figure defined and interpreted)
9. **Limitations**

If you only want to learn DIABLO, jump straight to `03_diablo/`. The folder is self-contained.

## Getting the data

You need the data once. The file sizes total ~186 MB. We recommend a direct download with `curl` over `git clone`-ing the upstream Hugging Face dataset, because:

- You only need five files (the GS-BRCA "Top" feature subset). Cloning brings the whole MLOmics archive.
- A `git clone` of the Hugging Face dataset requires `git-lfs`, which is not installed by default.
- Direct download is more reproducible: you get exactly the files you need, by exact URL.

```bash
mkdir -p data/raw/GS-BRCA/Top
BASE="https://huggingface.co/datasets/AIBIC/MLOmics/resolve/main/Main_Dataset/Classification_datasets/GS-BRCA/Top"
for f in BRCA_label_num.csv BRCA_miRNA_top.csv BRCA_mRNA_top.csv \
         BRCA_Methy_top.csv BRCA_CNV_top.csv; do
  curl -fL -o "data/raw/GS-BRCA/Top/$f" "$BASE/$f"
done
```

If you prefer `git clone` regardless, the dataset chapter (`01_dataset/dataset.qmd`) shows you that path too.

## Reproducing the tutorials

```bash
# Clone this repository
git clone https://github.com/derebellon/Methods_for_multiomics_integration.git
cd Methods_for_multiomics_integration

# Install R dependencies (CRAN + Bioconductor; idempotent)
Rscript R/install_deps.R

# Download the dataset (~186 MB; one-off)
bash scripts/download_data.sh

# Render the full website (the first render takes 30-60 minutes because the
# tuning chunks in DIABLO and MOFA2 are computationally heavy; cached
# renders after that take seconds per chapter).
quarto render
```

A locked `renv.lock` is **not** included in this initial release because
the heavier Bioconductor packages (MOFA2, WGCNA) compile from source on
Windows and the snapshot benchmarking is still in progress. We will
publish `renv.lock` in a follow-up release once the full render has been
validated on a clean machine.

The rendered website appears in `_site/`. Open `_site/index.html` in any browser. To rebuild only one tutorial:

```bash
quarto render 03_diablo/diablo.qmd
```

## How this repository was built

The tutorials and the supporting code were drafted iteratively by **David E. Rebellon-Sanchez** in collaboration with **Anthropic's Claude Opus 4.7** (1 M-context model), used as a writing and pair-programming assistant. Every code chunk follows the **canonical API documented by the original library authors**, with no custom wrappers or exotic variants, so that you can copy any chunk straight into your own project unchanged. Every interpretation has been read against the rendered output, and every metric is defined the first time it appears. All scientific claims are referenced to the primary literature in `references.bib`.

The intellectual responsibility for the content, including any errors, rests with the author.
